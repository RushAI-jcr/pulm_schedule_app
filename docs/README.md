# Documentation Map

This folder contains product planning, execution backlog, and operational runbooks.

## Structure

- `architecture/`: domain and technical architecture notes
- `sprints/`: sprint timeline and release milestones
- `backlog/`: ticketized epics and story-level acceptance criteria
- `runbooks/`: local dev, deployment, and disaster recovery procedures
- `governance/`: repository conventions and folder strategy
